Arbeidsmetodikk

Henrys filosofier

#1 — Abstraksjon

«En god abstraksjon gjør noe enklere å forstå. Ikke bare… abstrakt.»

#2 — Progressive enhancement

«Alle skjemaer skal fungere uten client-side JavaScript.»

#3 — Den indre kampen

«React Router v7 + SSR gjør jo .NET overflødig?… men. Ahmed er i rommet.»

Forbudte ting:

  • AutoMapper → manuelle extension-metoder
  • Repository-mønster → direkte EF DbContext
  • Global state / Redux
  • useState for forms → native HTML forms
  • modal (anti-pattern — se Invoker Commands API)
  • Swagger → Scalar
  • i18next → business language er norsk
  • ESLint + Prettier → Biome (10–100× raskere)
  • Next.js → «for opinionated»
